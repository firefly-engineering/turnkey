# Native Tool Wrappers for Auto-Sync

Turnkey automatically wraps native language tools (`go`, `cargo`, `uv`) to keep dependency files in sync when you use standard workflows.

## The Problem

When you run `go get github.com/foo/bar`, Go updates `go.mod` and `go.sum`. But Buck2 needs `go-deps.toml` (generated by `godeps-gen`) to know about the new dependency. Without auto-sync, you'd need to manually run:

```bash
go get github.com/foo/bar
godeps-gen --go-mod go.mod --go-sum go.sum -o go-deps.toml
```

This is tedious and easy to forget.

## The Solution

Turnkey transparently wraps `go`, `cargo`, and `uv` so that dependency sync happens automatically:

```bash
go get github.com/foo/bar  # Just works - go-deps.toml is auto-updated
```

## How It Works

```
┌─────────────────────────────────────────────────────────────────┐
│  User runs: go get github.com/foo/bar                           │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  Shell wrapper (provides 'go' binary)                           │
│  • Sets TURNKEY_REAL_GO to actual go binary path                │
│  • Calls: tw go get github.com/foo/bar                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  tw (turnkey wrapper)                                           │
│  1. Loads .turnkey/sync.toml configuration                      │
│  2. Finds wrapper rule for 'go'                                 │
│  3. Checks if 'get' is a mutating subcommand → yes              │
│  4. Captures SHA256 hashes of go.mod, go.sum                    │
│  5. Runs the real 'go get' command                              │
│  6. Compares hashes - detects changes                           │
│  7. Runs godeps-gen to regenerate go-deps.toml                  │
└─────────────────────────────────────────────────────────────────┘
```

## Configuration

### Enabling/Disabling Wrappers

Wrappers are **enabled by default**. To disable:

```nix
# In your flake.nix
turnkey.toolchains = {
  wrapNativeTools = false;  # Use unwrapped tools
};
```

### Wrapper Rules

Wrapper behavior is configured in `.turnkey/sync.toml`:

```toml
# Which commands trigger sync (generated by turnkey)
[[wrappers]]
name = "go"
command = "go"
mutating_subcommands = ["get", "mod"]
watch_files = ["go.mod", "go.sum"]
deps_rule = "go"

[[wrappers]]
name = "cargo"
command = "cargo"
mutating_subcommands = ["add", "remove", "update"]
watch_files = ["Cargo.toml", "Cargo.lock"]
deps_rule = "rust"

[[wrappers]]
name = "uv"
command = "uv"
mutating_subcommands = ["add", "remove", "lock", "sync"]
watch_files = ["pyproject.toml", "uv.lock"]
deps_rule = "python"
```

**Fields:**
- `command`: The tool being wrapped
- `mutating_subcommands`: Subcommands that might change dependency files
- `watch_files`: Files to monitor for changes (compared before/after)
- `deps_rule`: Name of the `[[deps]]` rule to trigger on change

### Deps Rules

The wrapper triggers the corresponding deps rule:

```toml
[[deps]]
name = "go"
sources = ["go.mod", "go.sum"]
target = "go-deps.toml"
generator = ["godeps-gen", "--go-mod", "go.mod", "--go-sum", "go.sum", "--prefetch"]
```

## Escape Hatches

### Bypass for a Single Command

```bash
TURNKEY_NO_WRAP=1 go get github.com/foo/bar
```

This runs the real `go` directly, skipping tw entirely.

### Disable Sync for a Command

```bash
tw --no-sync go get github.com/foo/bar
```

This runs through tw but skips the sync step even if files change.

### Verbose Output

```bash
tw -v go get github.com/foo/bar
# or
tw --verbose go get github.com/foo/bar
```

Shows what tw is doing:
```
tw: capturing state of [go.mod go.sum]
tw: detected changes in [go.mod go.sum], running sync
Syncing go-deps.toml...
  Running: godeps-gen --go-mod go.mod --go-sum go.sum --prefetch
  Regenerated go-deps.toml
```

## Supported Tools

| Tool | Mutating Commands | Watch Files | Deps Target |
|------|-------------------|-------------|-------------|
| `go` | `get`, `mod` | `go.mod`, `go.sum` | `go-deps.toml` |
| `cargo` | `add`, `remove`, `update` | `Cargo.toml`, `Cargo.lock` | `rust-deps.toml` |
| `uv` | `add`, `remove`, `lock`, `sync` | `pyproject.toml`, `uv.lock` | `python-deps.toml` |

## Adding Support for New Tools

1. **Add wrapper script** in `nix/packages/tw-wrappers.nix`:

```nix
tw-newtool = mkWrapper { name = "newtool"; pkg = pkgs.newtool; };
```

2. **Add to wrappable tools** in `nix/flake-parts/turnkey/default.nix`:

```nix
wrappableTools = [ "go" "cargo" "uv" "newtool" ];
```

3. **Add wrapper rule** to sync.toml generation (or manually):

```toml
[[wrappers]]
name = "newtool"
command = "newtool"
mutating_subcommands = ["install", "update"]
watch_files = ["newtool.lock"]
deps_rule = "newtool"
```

4. **Add deps rule**:

```toml
[[deps]]
name = "newtool"
sources = ["newtool.lock"]
target = "newtool-deps.toml"
generator = ["newtool-deps-gen", "-o", "newtool-deps.toml"]
```

## Troubleshooting

### "tw: no wrapper rule for X"

The tool isn't configured in `.turnkey/sync.toml`. Either:
- Add a `[[wrappers]]` section manually
- Ensure the turnkey devenv module is generating it

### "tw: no project root found"

tw looks for `.buckconfig` or `.turnkey/sync.toml` to find the project root. Ensure one exists.

### "tw: deps rule X not found"

The wrapper references a deps rule that doesn't exist. Add the corresponding `[[deps]]` section.

### Sync runs but nothing happens

Check that the generator command is correct:
```bash
tw -v go get github.com/foo/bar
```

The output shows the exact command being run.

## Implementation Details

### Change Detection

tw uses SHA256 content hashing (not mtime) to detect changes. This is more reliable across different filesystems and tools that may preserve timestamps.

### Signal Handling

tw properly forwards signals (SIGINT, SIGTERM) to the wrapped tool, so Ctrl+C works as expected.

### Exit Codes

tw preserves the exit code of the wrapped tool. If `go get` fails, tw exits with the same code.

### Non-Mutating Commands

Commands not in `mutating_subcommands` pass through without any overhead:

```bash
go build ./...    # No snapshot, no sync check - just runs go build
go version        # Direct passthrough
```
